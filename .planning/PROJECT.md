# 📋 PROJECT BRIEF — ВТОРНИК

## 🎯 Описание проекта

**ВТОРНИК** — литературно-художественный журнал на Vue 3 + Vite + Pinia

**Миссия:** Создание современной цифровой платформы для публикации литературных произведений с удобным CMS для редакторов и красивым интерфейсом для читателей.

---

## 📊 Текущий статус

**Фаза:** 1 — Стабилизация и доработка

**Последнее обновление:** 27 марта 2026

**Что готово:**
- ✅ Базовая структура проекта (Vue 3 + Vite + Pinia)
- ✅ 6 публичных страниц (Home, Issue, Article, Authors, AuthorDetail, About)
- ✅ 7 CMS страниц (Dashboard, Issues, Articles, Authors, Media, Prepare, Spine)
- ✅ Медиа-менеджер с иерархией папок
- ✅ Rich Text Editor с загрузкой изображений
- ✅ PostgreSQL + PostgREST (локально через Docker)
- ✅ Дизайн-токены + neuromorphic тема

**Что требует работы:**
- ⚠️ Данные в JSON (не сохраняются между сессиями)
- ⚠️ Нет аутентификации для CMS
- ⚠️ PDF генерация требует доработки
- ⚠️ Нет импорта данных со старого сайта

---

## 🎯 Цели проекта

### Приоритет 1 — Критично
1. **Интеграция с БД** — полное переключение на PostgreSQL через PostgREST
2. **Аутентификация** — защита CMS маршрутов через Supabase Auth
3. **Импорт данных** — перенос контента с vtornik.online

### Приоритет 2 — Важно
1. **PDF генерация** — подготовка к печати номеров
2. **Оптимизация** — lazy loading, code splitting
3. **404 страница** — обработка несуществующих маршрутов

### Приоритет 3 — Желательно
1. **Улучшения UX** — автосохранение, уведомления
2. **Расширение CMS** — массовые операции, история изменений
3. **Аналитика** — счётчик просмотров, статистика

---

## 🏗️ Архитектурные принципы

1. **Модульность** — разделение на компоненты, stores, services
2. **Composition API** — `<script setup>` для всех компонентов
3. **CSS Custom Properties** — дизайн-токены в variables.css
4. **API-first** — все данные через database.js сервис
5. **Progressive Enhancement** — работа без JS (базовая), улучшение с JS
